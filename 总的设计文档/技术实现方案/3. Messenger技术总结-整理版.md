## 一、项目基础定位与开发目标

### 1.1 开发初衷

| 维度 | 说明 |
|------|------|
| **创意目标** | 打造「平静又带一丝怪诞」的小型球形世界，参考日本城郊街巷风貌，无强制主线压迫，纯散步送信休闲体验 |
| **技术实验目标** | 自研轻量化多人 Web3D 环境技术，作为 15 分钟免费实验性 Demo 验证自研管线，实现全浏览器、手机低配设备流畅运行 |
| **核心世界设定** | 微型球形星球，任意方向直行可绕回起点；分为 7 大独立区域：**居民区、广场、墓地、海滩、山间寺庙、森林、工厂** |

### 1.2 底层硬性约束（浏览器优先）

全程基于原生 **WebGL**，**不使用 Unity/Unreal WebGL 打包**，规避臃肿运行时、超大包体、iOS Safari 内存崩溃问题；所有技术设计围绕浏览器沙盒内存、带宽、单线程限制优化。

## 三、核心难点：微型球形星球实现方案

### 方案 1：程序化模型曲面变形（❌ 废弃）

- 逻辑：普通建筑、地形模型加载后实时形变贴合球面
- 缺陷：物体摆放、细节修改成本极高，迭代效率极低 → 团队放弃

### 方案 2：「展开立方体映射星球」（✅ 最终方案，官方核心创新）

```
立方体平面建模（Blender） → 运行时顶点着色器球面映射（GLSL） → 球形星球
```

#### 建模阶段
- 以立方体 **6 个平整平面** 作为基底
- 在平面上手工搭建完整 7 区块场景
- 建筑、树木、道路全部在平整 UV 平面制作，**无球面扭曲**
- 建模、修改、细节添加效率极高

#### 运行阶段
- 浏览器渲染阶段，通过 GLSL 数学公式将立方体所有顶点坐标包裹弯曲为球形
- 全程仅代码计算，Blender 模型无需改动

#### 配套三大球形专属自研系统

| 系统 | 说明 |
|------|------|
| **动态相机系统** | 适配玩家在球体任意角度的跟随逻辑，自动规避山体、建筑碰撞，第三人称过肩视角 |
| **全局朝向校正** | 着色器自动修正树木、文字、NPC，保证曲面任何位置物体保持直立，不随球面倾斜翻转 |
| **星球中心重力物理** | 自定义物理系统，所有物体持续向星球核心产生引力，实现球面行走、不掉落 |

## 四、美术全流程管线

### 4.1 低模建模硬性标准（为浏览器性能定制）

全部手工低模白盒 → 细化，严控三角面数量：

- 单棵植被：简化 Blob 基础模型，自动替换优化面片植被几何体
- 单建筑、山体分段拆分，避免超大单网格；远景 LOD 极致减面
- 玩家全套角色（服饰、发型、配饰）低多边形模块化拆分，方便换装随机化
- 植被、路灯等重复道具使用 `InstancedMesh` 批量渲染，减少 DrawCall

### 4.2 LOD 细节层级系统（官方 4 级自定义 LOD）

1. **LOD**：Level of Detail，自研 **4 档梯度切换**，弱化模型切换"闪烁"观感
2. **制作流程**：
   - Blender 复制原始精细模型，使用 `Decimate` 减面修改器生成远距离简模
   - 同一物体精细模 + 简模绑定父子层级，统一导出至单个 GLB
   - 前端全局一套距离判断代码，所有建筑/NPC/树木自动切换对应精度模型
3. **效果**：近距离完整细节，远距离大幅删减三角面，手机低配设备稳定帧率

### 4.3 Draco 网格压缩（模型体积压缩核心）

| 项目 | 说明 |
|------|------|
| 性质 | 谷歌开源网格压缩算法，Blender 导出 GLB 时直接开启 |
| 作用 | 模型网格体积降低 **60% ~ 80%**，大幅缩短网页加载时间 |
| 配套 | 前端引入 Three.js Draco 解码器，浏览器加载时自动解压网格数据 |

### 4.4 16×16 全局色板（Color Atlas，画面色彩底层核心）

**制作**：Aseprite 新建 16px × 16px 画布，最大容纳 256 种固定纯色

**设计规则**：
- 游戏内天空、草地、建筑、人物、阴影 **全部复用这一张极小纹理**
- 无独立物体贴图、无 PBR 材质、无渐变纹理

**技术逻辑**：
```
模型存储色板索引值 → GLSL 着色器读取索引 → 16×16 色板查表 → 输出对应纯色
```

**工程优势**：
- 全局切换色调、昼夜氛围仅替换单张小贴图，无需修改所有场景模型
- 显存占用极低，规避 iOS 内存溢出闪退

### 4.5 场景细分特色美术系统

| 系统 | 实现方式 |
|------|----------|
| **智能植被** | 低模块状植被基底，着色器自动替换风格化简化草、树叶面片，批量渲染无性能压力 |
| **动态水体着色器** | 岸边波纹、水深渐变分层；玩家入水后衣物着色索引自动切换湿色，上岸缓慢恢复原色，**纯着色器逻辑，无额外动画资源** |
| **手绘轮廓描边管线** | 分两套方案：① 地形/静态道具 → 背面膨胀黑色描边（低成本）；② 角色 → 全屏 Sobel 边缘检测后处理精细描边；低配设备自动关闭全屏后处理 |

## 五、自定义 GLSL 着色器体系

> 项目核心难点，全场景所有物体共用两份基础代码，无需为建筑/NPC 单独编写。

### 5.1 顶点着色器

- 实现立方体 → 球面坐标映射
- 球形世界物体直立朝向校正
- LOD 模型顶点位置变换
- 膨胀描边模型顶点偏移

### 5.2 片元着色器

- 16×16 色板索引查表取色
- 3 档固定明暗（亮部 / 固有色 / 阴影，无实时光照渐变）
- 水体干湿变色
- 手绘轮廓线条计算

> ⚠️ 无任何第三方美术渲染插件，全部管线自研。

## 六、WebAssembly（Wasm）文字渲染专项优化

| 项目 | 说明 |
|------|------|
| **痛点** | 传统 MSDF 字体渲染大量文字会阻塞浏览器主线程，球面多角度文字易模糊 |
| **解决方案** | 使用 C++ 编写字形生成逻辑，编译为 Wasm 模块 |
| **工作流** | Wasm 实时生成文字字形 → 直接交付 GPU 渲染 → 球面任意倾斜角度文字依旧清晰锐利 |
| **UI 策略** | UI 全部在 WebGL 画布内绘制，**完全舍弃 HTML DOM 界面**，杜绝 DOM 与 3D 画布渲染抢占资源卡顿 |

## 七、浏览器极致内存 & 性能优化

> 重点针对 iPhone Safari

### 7.1 内存管控

| 策略 | 说明 |
|------|------|
| 全资产生命周期追踪 | 相机视野外的模型、贴图、音效立即从显存/内存完整销毁，无闲置占用 |
| 纹理极致轻量化 | 仅保留 16×16 色板单张核心纹理，无 4K 高清场景贴图 |
| 音效对象复用池 | 不重复创建音频实例 |

### 7.2 加载优化（点开即玩）

| 策略 | 说明 |
|------|------|
| 分块流式加载 | 开局仅加载星球基底 + 玩家模型，远处区块走到对应区域异步下载资源 |
| CDN + 强缓存 | 静态资源 CDN 托管，二次打开直接读取本地缓存 |
| PWA 离线缓存 | 支持手机添加桌面图标，离线可重复游玩 |

### 7.3 GPU 渲染减负

| 策略 | 说明 |
|------|------|
| `InstancedMesh` | 批量渲染树木、路灯、小草等重复道具，单次绘制上千实例，大幅减少 DrawCall |
| `three-mesh-bvh` | 包围盒视锥剔除，相机外物体直接跳过渲染 |
| 自动设备分级画质 | 代码检测设备性能，低配自动关闭描边、水体精细波纹、粒子特效 |

### 7.4 主线程减负

物理、角色坐标同步、换装更新逻辑分流至 **WebWorker 子线程**，不阻塞画面渲染循环。

## 八、多人联机轻量化同步系统

### 8.1 房间设计

- 单虚拟房间上限 **10 名玩家**
- 刻意控制同屏模型数量，维持游戏安静松弛氛围（技术层面同步压力同步降低）

### 8.2 极简同步数据包设计

| 设计 | 说明 |
|------|------|
| **无文字聊天** | 彻底删除文字聊天，自研 3D 悬浮 Emoji 表情交互系统，无需内容审核、极低网络传输量 |
| **最小化状态同步** | 仅同步：坐标、旋转朝向、当前表情 ID、换装外观 ID，单条数据包体积 **＜ 100 字节** |
| **同步频率** | 15 次/秒，休闲场景完全够用，不占用带宽 |

### 8.3 角色换装平滑更新机制

玩家更换服饰、发型会瞬时加载大量模型数据，易造成画面卡顿 → 团队自研 **"分段流式更新"系统**，分批次推送换装数据，全程帧率稳定无抖动。

### 8.4 网络防瞬移 & 断线恢复

| 能力 | 说明 |
|------|------|
| 坐标插值平滑 | 通过速度向量缓冲修正网络延迟导致的角色瞬移 |
| 断线自动重连 | 切页面、断网后恢复房间、玩家位置与外观状态 |

## 九、角色 & NPC 交互系统

### 9.1 玩家主角

- 以荷兰兼职送货青少年为原型
- 模块化拆分：头、上衣、下装、耳机、挎包
- 外观随机化，实现千人千面
- 基础动画仅保留：待机、行走、小幅跳跃（精简动画资源）

### 9.2 NPC 路人系统

| 设计 | 说明 |
|------|------|
| 统一低模人形规范 | 共用一套角色控制器代码 |
| NPC 能力 | 基础巡逻 AI、气泡文字对话、3D 空间定向音效 |
| 复用设计 | 全部 NPC 复用同一套移动、碰撞、动画逻辑，新增 NPC 仅配置坐标、外观参数，无需单独编写代码 |

### 9.3 全平台适配交互

| 平台 | 交互方式 |
|------|----------|
| **移动端** | 单指触屏完整操控，内置虚拟摇杆 |
| **PC 端** | 纯鼠标 + WASD 键鼠控制 |
| **相机** | 智能自动跟随，自动规避狭窄街巷墙体碰撞，新手零教程即可上手 |
